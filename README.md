# PvPBot Mod (Fabric)

Bukkit/Paper 플러그인으로 만들었던 PvP 봇을 Fabric 모드로 포팅하는 프로젝트입니다.
**지금은 아직 빈 뼈대 상태**이고, 이게 GitHub Actions에서 정상 빌드되는지부터 확인한 뒤
기능을 하나씩 이식할 예정입니다.

## 왜 뼈대부터 만들었나
Bukkit 플러그인 때는 "스텁 컴파일 기법"으로 로컬에서 실제 라이브러리 없이도 문법/타입 검증을
할 수 있었습니다. 그런데 Fabric 모드는:
- 마인크래프트 내부 클래스(엔티티, AI 등)에 직접 접근해야 하는데, 이건 안정된 공개 API가 아니라
  버전마다 이름이 바뀌는 내부 구현이라 스텁으로 흉내내는 게 사실상 불가능합니다.
- 이 샌드박스는 Fabric의 메이븐 저장소(`maven.fabricmc.net`)나 마인크래프트 배포 서버에 접근할
  네트워크 권한이 없어서, 실제 Loom 개발 환경을 로컬에서 구축해 컴파일 검증하는 것 자체가 불가능합니다.

그래서 이번엔 **GitHub Actions(진짜 인터넷 접근 가능한 환경)에서 직접 빌드해보면서 검증**하는
방식으로 갑니다. 뼈대 → 빌드 성공 확인 → 기능 하나씩 추가 → 빌드 확인, 이 순서로 진행하는 걸
추천드려요.

## 버전 정보 (2026년 7월 기준 최신 확인)
마인크래프트가 **26.1부터 최초로 비난독화(unobfuscated)** 버전이 됐습니다 (서버가 26.1.2이므로
정확히 이 버전대에 해당). 이게 Fabric 개발 방식에 큰 영향을 줍니다:

| | 1.21.11 이하 (난독화) | 26.1+ (비난독화, 지금 이 프로젝트) |
|---|---|---|
| Loom 플러그인 ID | `net.fabricmc.fabric-loom-remap` | `net.fabricmc.fabric-loom` |
| 매핑(mappings) | Yarn 필요 | **필요 없음** (Mojang 공식 이름 그대로) |
| 의존성 키워드 | `modImplementation` 등 | 그냥 `implementation` |
| 최종 산출물 태스크 | `remapJar` | 그냥 `jar` |

이 프로젝트는 26.1.2 기준으로 설정했습니다. 만약 서버를 1.21.11 이하로 쓰신다면 완전히 다른
설정(Yarn 매핑 포함)이 필요해요 — 말씀해주시면 그 버전용으로 다시 만들어드릴게요.

## 사용한 버전 (gradle.properties)
- Minecraft: `26.1.2`
- Fabric Loader: `0.18.4`
- Fabric Loom: `1.15.+`
- Fabric API: `0.155.2+26.2`
- Java: 25 (마인크래프트 26.1.2 자체가 JDK 25 기준이라 그보다 낮으면 안 됨)

**정직하게 밝혀둘 점**: 이 버전 조합을 웹 검색으로 확인은 했지만, 실제로 이 조합으로 빌드가
성공하는지는 로컬에서 검증할 방법이 없었습니다 (위 이유 때문에). GitHub Actions에서 처음
빌드해보시고 버전 안 맞는 에러가 나면 바로 대응할게요.

## 빌드
```
./gradlew build
```
GitHub에 push하면 `.github/workflows/build.yml`이 자동으로 빌드합니다.

## 로드맵 (앞으로 이식할 것들)
1. **이 뼈대가 빌드되는지 확인** (지금 여기)
2. 커스텀 몹 엔티티 등록 (Mannequin 대신 진짜 커스텀 엔티티나 바닐라 몹 상속)
3. AI 로직 이식 (거리 기반 상태머신 → 바닐라 `Goal`/`Brain` 시스템에 연결)
4. 무빙/경로탐색 (바닐라 `PathNavigation` 활용 검토 — 지금 플러그인의 10x10 BFS보다 훨씬 정교해질 수 있음)
5. 크리스탈 PvP, 물/불 생존, 채굴 등 나머지 전투 로직
6. 스킨 적용 (렌더링 파이프라인 직접 접근이라 플러그인 때보다 훨씬 확실하게 될 것으로 예상)

## 알려진 제한사항
- 지금은 정말 아무 게임 로직이 없습니다. `onInitialize()`에서 로그 한 줄만 찍습니다.
- Mixin 관련 설정(`mixins.json`, Mixin 의존성)이 아직 없습니다 — 실제 바닐라 코드에 훅을 걸려면
  다음 단계에서 추가해야 합니다.
- 클라이언트/서버 분리(`splitEnvironmentSourceSets`) 여부도 아직 정하지 않았습니다 — 스킨 렌더링
  같은 클라이언트 전용 기능을 넣으려면 이걸 고려해야 할 수 있습니다.
